# Orizum様向け インフルエンサーDBプロジェクト KPI定義書

**バージョン:** 1.0
**作成日:** 2026年1月20日
**作成者:** Manus AI

## 1. 概要

本ドキュメントは、インフルエンサーDB運用・自動更新プロジェクトの成功を定量的・定性的に測定するための重要業績評価指標（KPI）を定義するものです。これらのKPIは、プロジェクトが目指す「営業活動の効率化」と「データ資産価値の向上」がどの程度達成されたかを可視化します。

## 2. プロジェクト目標とKPIの関連性

| プロジェクト目標 | 関連するKPI | 測定内容 |
| :--- | :--- | :--- |
| **1. 入力業務の高速化と標準化** | ・月間新規追加インフルエンサー数<br>・データ入力品質 | バイトの作業効率と、入力されるデータの精度を測定する。 |
| **2. 営業活動の効率化** | ・月間DM送信数（初回・追い）<br>・追い漏れ件数<br>・返信率 | 営業担当者がどれだけ効率的にアプローチできているかを測定する。 |
| **3. データ資産価値の維持** | ・自動更新成功率<br>・データ陳腐化率 | DBの鮮度と信頼性がどれだけ維持されているかを測定する。 |

## 3. KPI詳細定義

### 3.1. プロセス効率に関するKPI

| KPI名 | 計算式 | 目標値（月間） | データソース | 測定目的 |
| :--- | :--- | :--- | :--- | :--- |
| **月間新規追加インフルエンサー数** | `COUNT(期間内の新規追加行)` | **300件** | スプレッドシート | バイトによる候補者発見プロセスの生産性を測る。 |
| **月間DM送信数** | `COUNT(期間内のDM送信ログ)` | **300件** | スプレッドシート / Webアプリ | 営業チーム全体のアプローチ量を測る。初回と追いを分けて集計することも有効。 |
| **返信率** | `(期間内の返信あり件数 / 期間内の初回DM送信数) * 100` | **10%** | スプレッドシート / Webアプリ | アプローチの質と、ターゲット選定の精度を測る。 |
| **追い漏れ件数** | `COUNT(dm_statusが「追いDM対象」かつnext_followup_atが一定期間過去のレコード)` | **0件** | スプレッドシート / Webアプリ | システム化によって、対応すべきインフルエンサーへのアプローチが漏れていないかを確認する。 |

### 3.2. データ品質に関するKPI

| KPI名 | 計算式 | 目標値 | データソース | 測定目的 |
| :--- | :--- | :--- | :--- | :--- |
| **自動更新成功率** | `(期間内の更新成功件数 / 期間内の全更新対象件数) * 100` | **98%以上** | スプレッドシート / Webアプリ | 自動更新機能が安定して稼働しているかを測る。 |
| **データ入力品質** | `(必須項目が全て埋まっている行数 / 全行数) * 100` | **100%** | スプレッドシート | カスタムGPTとバイトの運用によって、データの一貫性が保たれているかを測る。 |
| **データ陳腐化率** | `COUNT(last_updated_atが一定期間以上前のレコード) / 全行数` | **5%未満** | スプレッドシート / Webアプリ | DB全体の鮮度が維持されているかを測る。 |

## 4. レポーティング

-   **頻度:** 月次
-   **形式:** 各KPIの目標値と実績値を比較するシンプルなダッシュボード形式のレポートを作成します。
-   **担当:** プロジェクト推進担当者が集計し、関係者に共有します。

将来的には、Webアプリ（Phase 2）にこれらのKPIを自動で集計・可視化するダッシュボード機能を実装することを推奨します。
