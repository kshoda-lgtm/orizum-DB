## 成果物の概要

### Phase 0: スプシ運用を固める

**目的:** 現行のスプレッドシートを改善し、営業が効率的に日次業務を回せる状態を即座に実現する。

**主な改善点:**
- 自由記述の「備考」欄を廃止し、連絡先情報（`contact_type`, `contact_value`）を構造化
- ジャンル、特徴、メモをカスタムGPTの出力項目として明確化
- 営業向けに4つのビュー（未DM、追いDM対象、返信あり、更新失敗）を用意
- 追いDMルール（5営業日後など）を明文化

**成果物:** スプレッドシート定義書

---

### Phase 1: YouTube自動更新

**目的:** DBのデータ鮮度を自動で維持し、営業が常に最新情報に基づいてアプローチできる環境を構築する。

**主な特徴:**
- 毎日深夜に自動実行される更新バッチ（Google Cloud Functions推奨）
- YouTube Data API v3を使用し、登録者数とジャンル情報を取得
- 更新結果（成功/失敗/理由）をスプレッドシートに記録
- 現在の1000行規模では、API無料利用枠内で運用可能

**成果物:** YouTube自動更新 設計書

---

### Phase 2: Webアプリ化

**目的:** 検索性と視認性を大幅に向上させ、営業活動をさらに加速させる。

**主な機能:**
- **ダッシュボード:** 担当者別の「今日のToDoリスト」を表示
- **検索・一覧:** ジャンル、登録者数、ステータスなど複数条件での検索
- **クリエイター詳細:** 基本情報、指標、接触履歴、AIによるDM提案を1画面に集約

**成果物:** Webアプリ 要件定義書

## 受入条件（プロジェクト完了の判定基準）

### Phase 0 完了時点

- ✅ バイトがカスタムGPTの出力を利用して、定義されたフォーマットでスプレッドシートにデータを追加できる
- ✅ 営業担当が「未DM」「追い対象」ビューを見るだけで日々のDM送信業務が完結し、追い漏れが構造的に発生しなくなる

### Phase 1 完了時点

- ✅ YouTubeチャンネルの登録者数が、人手を介さずに毎日自動で更新される

### Phase 2 完了時点

- ✅ 営業担当が、Webアプリの検索機能を用いて、複数の条件（ジャンル、規模、ステータス等）でインフルエンサーを絞り込める
- ✅ クリエイター詳細画面で、インフルエンサーの基本情報、接触履歴、次のアクション、AIによるDM提案が1画面で確認できる
