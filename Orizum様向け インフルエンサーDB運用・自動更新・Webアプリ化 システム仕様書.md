**バージョン:** 1.0

**作成者:** Manus AI

## 1. はじめに

### 1.1. プロジェクトの目的

本プロジェクトは、Orizum社におけるインフルエンサーマーケティングの営業活動を効率化し、データに基づいた持続的なアプローチを実現することを目的とします。具体的には、以下の3つの目標を達成します。

1.  **入力業務の高速化と標準化:** バイトが行うインフルエンサー情報の入力を、カスタムGPTを用いて高速化し、常に一定の品質を担保します。
2.  **営業活動の効率化:** 営業担当者が「次に誰に何をすべきか」を迷わず判断できる仕組みを構築し、アプローチの漏れや重複を削減します。
3.  **データ資産価値の維持:** 登録されたインフルエンサーの主要指標を自動で更新し、データベースが陳腐化することを防ぎます。

### 1.2. 本ドキュメントの位置づけ

本ドキュメントは、このプロジェクトで開発するシステムの全体像、アーキテクチャ、機能、運用フローを定義するものです。これまで作成した以下の設計書を統合し、プロジェクト全体の技術的な仕様を網羅します。

-   スプレッドシート定義書
-   YouTube自動更新 設計書
-   Webアプリ 要件定義書

## 3. 人・システム・AIの役割分担

| 担当 | 役割 | 具体的なタスク |
| :--- | :--- | :--- |
| **人（バイト）** | **候補者の発見と一次入力** | ・カスタムGPTを使いインフルエンサー候補をリストアップ<br>・目視で最終判断し、スプレッドシートに情報をペースト |
| **人（営業）** | **意思決定とコミュニケーション** | ・誰に、いつ、どの案件でアプローチするかの最終判断<br>・DMの送信、および返信への対応 |
| **システム** | **データ管理とプロセスの自動化** | ・DBの正規化、ステータス管理、更新ログの記録<br>・YouTube指標の自動更新<br>・営業向けキュー（未DM、追い対象など）の提示 |
| **AI** | **リサーチと提案による補助** | ・バイト向けのリサーチと定型フォーマットでの情報出力<br>・営業向けのDM文案提案（複数案＋根拠） |

## 4. データモデル

システムの中心となるデータモデルは、スプレッドシート定義書で定義した通りです。Webアプリ化（Phase 2）の際は、`Creators` テーブルと `Outreach`（接触履歴）テーブルに分割して管理することを推奨します。

-   **主要エンティティ:** `Creators` (インフルエンサーマスタ)
-   **主キー:** `channel_url`
-   **詳細:** [スプレッドシート定義書](./spreadsheet_definition.md) を参照。

## 5. 機能仕様

### 5.1. Phase 0: スプシ運用

-   **列定義:** 詳細は[スプレッドシート定義書](./spreadsheet_definition.md)を参照。
-   **ビュー定義:** 「未DM」「追いDM対象」「返信あり」「更新失敗」の4つのビューを用意し、営業の日次業務を支援する。

### 5.2. Phase 1: YouTube自動更新

-   **アーキテクチャ:** Google Cloud Functions + Cloud Scheduler + YouTube Data API v3
-   **更新対象:** 登録者数、チャンネルジャンル
-   **実行頻度:** 毎日深夜
-   **詳細:** [YouTube自動更新 設計書](./youtube_auto_update_design.md) を参照。

### 5.3. Phase 2: Webアプリ

-   **主要画面:** ダッシュボード、検索・一覧、クリエイター詳細
-   **主要機能:** 担当者別キュー表示、複合条件検索、接触履歴タイムライン、AIによるDM提案
-   **データ同期:** 初期はGoogle Sheets API経由でスプレッドシートを直接参照。将来的には専用DBへ移行。
-   **詳細:** [Webアプリ 要件定義書](./web_app_requirements.md) を参照。

## 6. 受入条件

本プロジェクトの完了は、以下の条件が満たされたことを以て判断します。

| フェーズ | 受入条件 |
| :--- | :--- |
| **Phase 0** | ・バイトがカスタムGPTの出力を利用して、定義されたフォーマットでスプレッドシートにデータを追加できる。<br>・営業担当が「未DM」「追い対象」ビューを見るだけで日々のDM送信業務が完結し、追い漏れが構造的に発生しなくなる。 |
| **Phase 1** | ・YouTubeチャンネルの登録者数が、人手を介さずに毎日自動で更新される。<br>・更新に失敗した行がある場合、その理由がスプレッドシート上で可視化され、手動での対応が可能になる。 |
| **Phase 2** | ・営業担当が、Webアプリの検索機能を用いて、複数の条件（ジャンル、規模、ステータス等）でインフルエンサーを絞り込める。<br>・クリエイター詳細画面で、インフルエンサーの基本情報、接触履歴、次のアクション、AIによるDM提案が1画面で確認できる。 |
